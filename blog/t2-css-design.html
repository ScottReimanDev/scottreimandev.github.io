Classes vs IDs

How Do I Know Which to Use?

11-30-2014

When you are writing HTML and CSS code, sometimes you need to style certain pieces of your HTML, but not others. For example, you may have a several lists you are making. What if you wanted to format one list a certain way, but you wanted to format all of the other lists a different way? This is where classes and IDs come into play. A class or an ID will allow you to reference that specific element, and then you can style it however you want, even if it is different from the rest of the elements. If you attach a class or an ID to an element in HTML, you can reference that specific element or group of elements in CSS simply by recalling that class or ID. Make sense? I didn't think so. Let's get into the difference between classes and IDs, and hopefully it will all start to come together.

We'll start with classes. A class tag can be added to any element. Classes must be given identifiers so that you can reference this when adding your CSS code. Let's say I am making a website, and I want to list blog posts that I have created by the week in which they were written. I can make tags for the individual weeks, but what if I want to style these tags? I can underline them, put them in bold, italics, etc. If I want to style all of these tags the same, then I need to add a class. By adding the same class to each tag, I can style them all at the same time by referring to that class in my CSS code. Let's look at an example:

Since I want to style all three of those headers the same, I can assign them a class (in this case, I called the class "weeks"), and then I can reference that class to style them all at the same time. To reference a class in CSS, you use a period followed by the class name. In this case it would be ".weeks". Again look at this example:

You can see that in our CSS code, we have referenced the class "weeks" by typing ".weeks". This opens up our brackets {}, and inside we can style every element that has the class "weeks." You can see how the labels "Week 1", "Week 2" and "Week 3" are all underlined and in blue now. This was only a list of three things, but you can imagine how helpful using classes can be if we have many more items that we want to style all the same way. Instead of having to style each individual element, with classes we can style them all at once.

So if those are classes, then what are IDs? IDs are very similar to classes in that they can be used to style individual HTML elements in CSS. Unlike classes, though, IDs are unique to the element they get attached to. If you only have one element that you want to style a certain way, then you can give it an ID. So there you have it. To sum up, classes are used to style a group of elements the same way, while IDs are used to style a single element a certain way. Using classes and IDs can be a huge time-saver if you know how to use them correctly, and hopefully after reading this blog you are familiar enough with them to give them a shot on your own!
